Linux Provider

Let’s add a HANA Provider to collect metrics for us:

Node Exporter Agent

Before we start, we need to add an agent to the monitored VM.

  1. Go to the Azure Portal and open a Cloud Shell. In the examples we used BASH as the interpreter.
  2. Now we will upload the key that SAP CAL generated on the deployment so we can log on to the Linux OS. Click on the File Transfer icon and then Upload
  3. Select the key which will be a *.pem file, downladed from SAP CAL. If you do not remember or cannot find the file, go to SAP CAL, select your instance, click on Download Key and download it again.
  4. Let’s add permissions to the .PEM file uploaded on the Azure Cloud Shell (in this example the key is called demosap.pem)
chmod 400 demosap.pem
  1. With the key setup done, we can connect to the SAP HANA instance, usign the key as password for root:
ssh -i demosap.pem root@<PUBLIC IP of HANA>
  1. Let’s download the agent. Go to Prometheus Download Page and copy the address for the latest Node Exporter for Linux
  2. Back to Azure Cloud Shell, we will download, extract the agent and copy it to /usr/bin.

  1. On the Bastion Host, go to http://«SAP HANA Private IP»:9100/metrics
  2. Kill the node_exporter process with CTRL+C and start it in background
nohup node_exporter &

The agent is installed and communicating internally on the vNET. Let’s add the provider now

Linux Provider

  1. On the Azure Monitor for SAP, go to Providers
  2. Click Add
  3. Let’s fill the required data:
  • Type: OS Linux
  • Endpoint: http://internal_ip_address:9100/metrics
  1. Wait until the provider is created and give it 10-15 minutes for data to start to flow.
  2. Go to Monitoring, select OS (Linux), and pick the host.

Congratulations, you just installed and finished the OS Linux Agent install.
