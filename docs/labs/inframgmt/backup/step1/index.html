Setup Azure Backup

In order to have Azure Backup backing up SAP HANA databases, we need to setup the Azure Backup as well as the HANA Database Server.

This section will show the steps required for that to be accomplished:

SAP HANA Setup

We need to connect to the SAP HANA Server and run a script that will prepare the database for Azure Backup.

  1. Log on to the Azure Portal and open a Cloud Shell (BASH)
  2. We need to SSH to the server, using the SAP provided certificate key (can be downloaded from SAP CAL again), previously uploaded to the CloudShell and having permission as 400).
mod@Azure:~$ ssh -i <KEYNAME>.pem root@<HANA PUBLIC IP>
sid-hdb-s4h:~ #
  1. Then we will change the user to the HANA administrator hdbadm, store the database password in a secure storage, and return to root.
sid-hdb-s4h:~ # sudo su - hdbadm
sid-hdb-s4h:HDB:hdbadm /usr/sap/HDB/HDB02 2> hdbuserstore set azure_key localhost:30213 SYSTEM '<YOUR SAP DEFINED PASSWORD>'
sid-hdb-s4h:HDB:hdbadm /usr/sap/HDB/HDB02 4> exit
logout
sid-hdb-s4h:~ #

At this point our HANA database is ready to be accessed by Azure backup and Backint (SAP Native Backup Agent) has been configured to send data to Azure.

Azure Backup Setup

  1. Log on to the Azure Portal and open Backup Center
  2. On Backup Center we will create a Vault to store the backup data
  3. For Vault Type select Recovery Services Vault
  4. Provide the Vault details:
  • Region: East US (Same as HANA database deployment)
  1. Once the Vault is created, let’s configure the Backup targets inside the Vault.
  2. Change the backup type to SAP HANA on Azure VM and click on Start Discovery.
  3. Select the HANA VM «instance name-SAP1 and click on Discover DB at the bottom of the page
  4. Azure will start a deployment for the agent. Wait until you see the sucessfull message on Notifications
  5. Back to the Backup page, you should now have a View Details button. Click it so we can see the discovered Databases.
  6. Here you can see the discovered databases. Check the info and then close then window to go back to Backup
  7. Click on Configure Backup
  8. You can accept the default policy, select an existing one or Edit the current policy.
  9. In this lab we will create a simpler policy, called DemoPolicy. Click on Edit besides each item and configure it so it matches the example below.
  10. Now let’s add the discovered HANA databases. Click in Add and on the window that will open, select the DBs.
  11. Click on Enable Backup once you have selected the HDB and SYSTEMDB databases.
  12. Now, let’s make sure the items were correctly added and prepare for the first backup. Click on Backup items and then SAP HANA in Azure VM
  13. You should see a screen similar to this, with a Warning (initial backup pending) message

Congratulations, you have configured the HANA Backup using Azure Backup. Now let’s move to the next section and run our first manual backup.
